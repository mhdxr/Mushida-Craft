# Rencana: Vercel Deployment Setup

## Pendekatan
Vercel auto-detect Next.js project, jadi deploy utama tinggal push ke GitHub + import di Vercel Dashboard. Yang kita buat: config file untuk optimalisasi + dokumentasi lengkap.

## File yang akan dibuat/diubah

### 1. `vercel.json` (BARU)
- Framework: Next.js (auto-detected, tapi eksplisit)
- Build command: default (`next build`) — tidak perlu override
- Headers keamanan: X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy
- Redirect: domain apex → www (opsional, via comment)
- Tidak set env vars di sini (env vars di-set via Vercel Dashboard untuk keamanan)

### 2. `.vercelignore` (BARU)
- `supabase/` — migration SQL tidak perlu di-deploy
- `scripts/` — setup-db script hanya untuk local/CI
- `.env.example` — tidak perlu di-deploy
- `*.md` (kecuali README untuk GitHub) — dokumentasi tidak perlu

### 3. `README.md` (UBAH — tambah section deployment)
Tambah section "🚀 Deployment ke Vercel" dengan:
- Prasyarat (GitHub repo, Supabase project, optional Sentry)
- Step-by-step: import project di Vercel → set env vars → deploy
- Checklist env vars yang harus di-set di Vercel Dashboard
- Catatan: `npm run db:setup` dijalankan lokal sebelum deploy (atau via CI)
- Troubleshooting tips

## Yang TIDAK diubah
- `next.config.mjs` — sudah compatible (Sentry wrapper OK di Vercel)
- `package.json` — scripts sudah standard Vercel
- Env vars — tetap di Vercel Dashboard, bukan di vercel.json

## Verifikasi
- `npm run build` — pastikan tetap sukses dengan vercel.json present
